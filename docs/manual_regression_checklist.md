# Manual Regression Checklist: Document Flow + Google Drive

Дата: ____-__-__
Проверял: __________________
Сборка/ветка: __________________

## 1. Базовый жизненный цикл документа

- [ ] `Новый документ` создает пустой документ.
- [ ] В Header отображается `Новый документ`.
- [ ] После редактирования появляется `*` в Header у имени документа.
- [ ] После редактирования появляется `*` у иконки `Сохранить` в `CalendarIconBar`.

## 2. Сохранение

- [ ] `Сохранить` без открытого файла (`fileId == null`) показывает сообщение: использовать `Сохранить как...`.
- [ ] `Сохранить как...` открывает диалог и позволяет выбрать папку/имя.
- [ ] После успешного `Сохранить как...` имя документа в Header обновляется на имя сохраненного файла.
- [ ] После успешного `Сохранить как...` dirty-флаг (`*`) исчезает.
- [ ] После повторного изменения dirty-флаг снова появляется.
- [ ] `Сохранить` после `Сохранить как...` сохраняет в тот же файл и снимает dirty-флаг.

## 3. Открытие документов

- [ ] `Открыть из Google Drive` открывает пикер файлов.
- [ ] Выбор файла в пикере загружает его данные в приложение.
- [ ] Header показывает имя открытого файла.
- [ ] При выборе папки вместо файла загрузка не выполняется.

## 4. Открыть последний документ

- [ ] При наличии `lastOpenedDocument` и авторизации `Открыть последний документ` загружает документ.
- [ ] При отсутствии `lastOpenedDocument` показывается понятное сообщение.
- [ ] Если пользователь не авторизован, по клику открывается login-flow.
- [ ] После успешного логина документ загружается.

## 5. Новый/Закрыть документ + защита от потери

- [ ] При dirty и действии `Новый документ` показывается диалог `Сохранить / Не сохранять / Отмена`.
- [ ] Ветка `Отмена` отменяет действие.
- [ ] Ветка `Не сохранять` продолжает действие без сохранения.
- [ ] Ветка `Сохранить` сохраняет и затем выполняет действие.
- [ ] То же поведение подтверждено для `Закрыть документ`.
- [ ] То же поведение подтверждено перед `Открыть из Google Drive`.
- [ ] То же поведение подтверждено перед `Открыть последний документ`.

## 6. Конфликты Save As

- [ ] При `Сохранить как...` в папку с существующим именем показывается диалог конфликта.
- [ ] Ветка `Перезаписать` обновляет существующий файл.
- [ ] Ветка `Переименовать` сохраняет файл с новым именем.
- [ ] Ветка `Отмена` прерывает сохранение без побочных эффектов.

## 7. Ошибки и отказоустойчивость

- [ ] Открытие невалидного JSON показывает ошибку, приложение не падает.
- [ ] Ошибка доступа к Drive (403/404) отображается корректно.
- [ ] После ошибки UI остается рабочим (можно повторить действие).

## 8. Поведение после перезагрузки

- [ ] `lastOpenedDocument` сохраняется после `open/save/save as/new`.
- [ ] `lastOpenedDocument` очищается после `Закрыть документ`.
- [ ] При старте приложения не происходит авто-popup логина без клика пользователя.
- [ ] При авторизации пользовательским действием восстановление документа работает.

## 9. Локальное сохранение

- [ ] `Сохранить локально` сохраняет данные в localStorage.
- [ ] Индикатор локальной синхронизации обновляется корректно.

## 10. Визуальные и текстовые проверки

- [ ] Названия команд в меню и тултипах консистентны (`Сохранить`, `Сохранить как...`, `Открыть из Google Drive`, `Открыть последний документ`).
- [ ] Тексты ошибок/подсказок на русском и понятны пользователю.
