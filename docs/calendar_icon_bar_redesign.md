## 2. Принципы редизайна

### 2.1 Ключевые изменения в архитектуре:

1. **Автосохранение в localStorage** — данные сохраняются автоматически при каждом изменении
2. **Multi-document с вкладками** — пользователь работает с активным документом среди нескольких открытых
3. **Синхронизация по требованию** — Google Drive синхронизация только по явному запросу
4. **Визуальные индикаторы** — вкладки показывают статус (изменён, офлайн, синхронизирован)
5. **Безопасность данных** — НИКОГДА не перезаписывать данные автоматически при синхронизации

### 2.2 Что убираем:

- ❌ "Сохранить локально" — автосохранение работает автоматически
- ❌ "Открыть последний документ" — сессия восстанавливается при запуске
- ❌ "Сохранить как..." — избыточная функция, можно переименовать документ
- ❌ Дубли кнопок в меню

### 2.3 Что добавляем:

- ✅ "Синхронизировать с Google Drive" — явная синхронизация активного документа с диалогом выбора версии
- ✅ "Синхронизировать все документы" — массовая синхронизация офлайн-документов
- ✅ Индикаторы статуса на кнопках (badge с количеством)
- ✅ Диалог конфликта версий — пользователь выбирает, какая версия будет окончательной

```typescript
// ✅ Синхронизация активного документа с диалогом выбора версии
const handleSyncActiveDocument = async () => {
    if (!activeDoc?.ref?.fileId) {
        alert('Документ не связан с Google Drive')
        return
    }
    
    const result = await documentTabsStore.syncActiveDocumentWithDrive()
    
    if (result.status === 'conflict') {
        // Показать расширенный диалог выбора версии
        showConflictDialog(result)
    } else if (result.status === 'error') {
        alert(result.message)
    }
    // success — ничего не показывать, данные уже обновлены
}

// ✅ Синхронизация всех документов
const handleSyncAllDocuments = async () => {
    const results = await documentTabsStore.syncAllDocumentsWithDrive()
    
    let successCount = 0
    let errorCount = 0
    let conflictCount = 0
    
    results.forEach((result, id) => {
        if (result.status === 'success') successCount++
        else if (result.status === 'error') errorCount++
        else if (result.status === 'conflict') conflictCount++
    })
    
    if (errorCount > 0 || conflictCount > 0) {
        alert(`Синхронизация завершена: ${successCount} успешно, ${errorCount} ошибок, ${conflictCount} конфликтов`)
    }
}

// ✅ Переименовать документ
const handleRenameDocument = async () => {
    if (!activeDoc) return
    
    const newName = prompt('Введите название документа:', activeDoc.ref?.name)
    if (newName && newName.trim()) {
        documentTabsStore.updateActiveDocumentRef({
            ...activeDoc.ref,
            name: newName.trim()
        })
    }
}
```

```typescript
            return 'Ошибка синхронизации'

            return ''
    }
```

### 7.2 Сравнение версий документов

**Критерий сравнения:** Временная метка последнего изменения (`modifiedTime`).

| Параметр | Источник | Тип | Пример |
|----------|----------|-----|--------|
| `remoteModifiedAt` | Google Drive `modifiedTime` | ISO 8601 string | `"2026-03-14T10:30:00.000Z"` |
| `localModifiedAt` | DocumentSession `lastSavedAt` | Unix timestamp (ms) | `1710411000000` |
| `hasLocalChanges` | DocumentSession `isDirty` | boolean | `true` |

**Алгоритм проверки:**

```typescript
const remoteModifiedAt = remoteMetadata.modifiedTime
    ? new Date(remoteMetadata.modifiedTime).getTime()
    : 0

const localModifiedAt = session.state.lastSavedAt ?? 0
const hasLocalChanges = session.state.isDirty
const hasRemoteChanges = remoteModifiedAt > localModifiedAt

// === КЛЮЧЕВОЕ ПРАВИЛО: Показываем диалог при любых изменениях ===
if (hasLocalChanges || hasRemoteChanges) {
    return {
        status: 'conflict',
        hasLocalChanges,
        hasRemoteChanges,
        // ...
    }
}

// Нет изменений — загружаем для проверки целостности
const content = await this.googleApiService.downloadFileContent(fileId)
```

### 7.6 Массовая синхронизация всех документов

**Алгоритм:**

```typescript
async syncAllDocumentsWithDrive(): Promise<Map<DocumentId, SyncResult>> {
    const results: Map<DocumentId, SyncResult> = new Map()
    
    // Фильтрация: только документы с fileId и статусом 'offline'
    for (const [id, session] of this.state.documents.entries()) {
        if (session.ref?.fileId && session.state.syncStatus === 'offline') {
            // Временно активируем документ для синхронизации
            const previousActiveId = this.state.activeDocumentId
            this.state.activeDocumentId = id
            
            // Синхронизируем
            const result = await this.syncActiveDocumentWithDrive()
            results.set(id, result)
            
            // Восстанавливаем активный документ
            this.state.activeDocumentId = previousActiveId
        }
    }
    
    return results
}
```

**Статистика синхронизации:**

```typescript

results.forEach((result) => {
    if (result.status === 'success') successCount++
    else if (result.status === 'error') errorCount++
    else if (result.status === 'conflict') conflictCount++
})

// Уведомление пользователя
if (errorCount > 0 || conflictCount > 0) {
    alert(`
        Синхронизация завершена:
        ✅ Успешно: ${successCount}
        ❌ Ошибок: ${errorCount}
        ⚠️ Конфликтов: ${conflictCount}
    `)
}
```

### 7.7 Статусы синхронизации

| Статус | Значение | Когда устанавливается | Иконка |
|--------|----------|----------------------|--------|
| `offline` | Документ открыт из localStorage, синхронизация не выполнена | При восстановлении сессии / после редактирования | 📴 |
| `syncing` | Идёт процесс синхронизации | При вызове `syncActiveDocumentWithDrive()` | 🔄 |
| `synced` | Данные актуальны (локальная версия = Drive) | После успешной синхронизации | ✓ |
| `needs-sync` | Есть изменения для синхронизации | После обнаружения изменений (локальных или на Drive) | ⚠ |
| `error` | Ошибка при синхронизации | При ошибке сети / авторизации / доступа | ❌ |

**Переходы между статусами:**

```

│                    Диаграмма состояний                          │

                            ▼
```

## 10. Проверка безопасности данных

### Чек-лист защиты от потери данных:

- [ ] Локальные изменения (`isDirty`) никогда не перезаписываются автоматически
- [ ] Диалог показывается при наличии локальных изменений
- [ ] Диалог показывается при изменениях на Drive
- [ ] Пользователь явно выбирает версию при конфликте
- [ ] Можно сохранить локальную версию в Drive
- [ ] Можно загрузить версию с Drive
- [ ] Статусы синхронизации обновляются корректно
- [ ] Массовая синхронизация обрабатывает конфликты

### Критерии приёмки:

1. **Безопасность данных:** Локальные изменения никогда не перезаписываются автоматически
2. **Прозрачность:** Пользователь всегда видит состояние обеих версий
3. **Контроль:** Пользователь выбирает, какая версия будет окончательной
4. **Ясность:** Диалог понятно объясняет ситуацию и варианты действий
